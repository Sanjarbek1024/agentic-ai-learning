# AI Agent with PostgreSQL Memory & Tool (n8n)

## Overview

This project demonstrates how to build an AI-powered Telegram chatbot using **n8n**, **Groq LLM**, and **Neon PostgreSQL**.

The AI agent can:

- Answer user questions.
- Remember previous conversations using PostgreSQL Chat Memory.
- Generate SQL queries automatically.
- Read and modify data in PostgreSQL.
- Return database results in natural language.

---

## Technologies

- n8n Cloud
- Telegram Bot API
- Groq Chat Model (openai/gpt-oss-120b)
- Neon PostgreSQL
- PostgreSQL Chat Memory
- PostgreSQL AI Tool

---

## Architecture

```
Telegram User
      │
      ▼
Telegram Trigger
      │
      ▼
AI Agent
├── Groq Chat Model
├── PostgreSQL Chat Memory
└── PostgreSQL Tool
        │
        ▼
 Neon PostgreSQL
      │
      ▼
Telegram Response
```

---

## Workflow

1. The user sends a message through Telegram.
2. Telegram Trigger receives the message.
3. AI Agent processes the request.
4. Previous conversation is loaded from PostgreSQL Chat Memory.
5. If database access is required, the AI generates a PostgreSQL query.
6. PostgreSQL Tool executes the generated SQL.
7. Results are returned to the AI Agent.
8. AI Agent converts the result into a human-readable response.
9. The response is sent back to Telegram.

# AI Agent

The AI Agent is responsible for:

- Understanding user requests.
- Using PostgreSQL Tool when database information is required.
- Generating valid PostgreSQL SQL queries.
- Returning concise answers in the user's language.
- Never fabricating database information.

---

# PostgreSQL Chat Memory

Conversation history is stored in PostgreSQL.

Session Key:

```javascript
{{ $json.message.from.id }}
```

Each Telegram user has an independent conversation history.

---

# PostgreSQL Tool

The PostgreSQL Tool executes SQL generated by the AI.

Tool Description:

- Execute SQL queries on PostgreSQL.
- Use this tool whenever database information is required.
- Generate valid PostgreSQL SQL only.
- Never fabricate database results.

Query Expression:

```javascript
{{ $fromAI("query", "Valid PostgreSQL SQL query") }}
```

This allows the AI Agent to dynamically generate SQL queries during execution.

# Conclusion

This project demonstrates how to connect an AI Agent with a PostgreSQL database using n8n. The agent can understand natural language, generate SQL queries automatically, execute them against Neon PostgreSQL, and return clear, human-readable responses. By combining PostgreSQL Chat Memory with AI Tool Calling, the chatbot maintains conversation context while interacting with live database data. This project provides a practical introduction to AI agents, tool calling, memory management, and database integration within modern LLM workflows.
